# Performance Results

This directory is reserved for later baseline evidence. Phase A creates no latency or throughput results.

For each future run, create a timestamped directory containing:

- A completed run metadata record.
- One endpoint result record per measured endpoint.
- Raw k6 output when Phase B is authorized.
- Bounded SQL query-count evidence.
- PostgreSQL EXPLAIN output.

Do not store passwords, JWTs, JDBC bind values, personal data, or production data here.
